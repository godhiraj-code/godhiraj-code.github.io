If one has gone through the godoc , then they use the term command frequently . By command , they are saying executable (i.e main ) . So don't get confused.

Let's get straight to practical concepts that we need to know before we start our coding

WorkSpaces :
Go code needs to be kept inside a workspace .
workspace is a directory with the following sub directories

src - contains your go source files organized into packages
pkg- contains package objects
bin - contains executable commands

This is how a workspace should look like


workspace
